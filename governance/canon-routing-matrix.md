# EVE Canon Routing Matrix

## Purpose
Decide where each output belongs, what class it is, and what must happen before it becomes official.

## Routing law
- EVE = governance and approval
- PTR or Peter = workshop, founder doctrine, refinement proposals
- Nubeing 9 Studios = business, production, website, services, public deployment
- NeoSophia = canon, mythos, worldbuilding, symbolic systems, seeker experience
- Archive or reference = material that should be preserved but not treated as active doctrine

## Routing loop
1. Identify the output
2. Decide its project home
3. Decide whether it is exploratory, operational, canonical, or archive-only
4. Log it
5. Assign next action
6. Review later if it changes

## Plain-language rule
Nothing should stay floating. Everything gets a home.

## Quality law
Correct. Coherent. Clear. Precise.
