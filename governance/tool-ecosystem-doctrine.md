Approved and sealed by EVE as the current governance-approved tool stack.

## Governance intent
This document defines the active Premier tool tier, the active Bench tier, the sandbox doctrine, the open-source migration path, and the operating logic for how tools should be used, promoted, tested, or held in reserve.

## Governing principle
- Peter = workshop, refinement, proposal space
- EVE = governance, lock, approval, operating authority

From this point forward, refinements in Peter are treated as proposals, and no major Premier-tier change is treated as active doctrine unless confirmed by EVE.

## Open-source migration doctrine
Use practical paid tools where they currently outperform for speed, usability, and output.
Gradually build competence in open-source and sovereign alternatives.
Migrate only when an open alternative proves stable, aligned, and operationally strong.

Current migration anchors:
- Ollama
- OpenRouter
- GitHub
- OpenClaw

## Promotion and review logic
### Promotion rule
A Bench tool is only promoted if it:
1. performs the job better, or more appropriately, than the current Premier tool
2. is stable enough for repeated use
3. improves cost, control, quality, or sovereignty in a meaningful way

### Retirement or demotion rule
A Premier or Bench tool should be demoted, parked, or retired if it:
1. overlaps heavily with a stronger tool
2. creates friction without unique value
3. no longer fits the ecosystem’s working rhythm
4. is clearly outperformed by a validated alternative
