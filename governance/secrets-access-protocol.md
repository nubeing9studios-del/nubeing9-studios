# EVE Secrets and Access Protocol

## Purpose
Govern passwords, keys, tokens, permissions, recovery methods, and access reviews.

## Core rules
- Do not leave important credentials undocumented.
- Record owner, platform, recovery method, and storage location.
- Track who has access and why.
- Review sensitive access regularly.
- If a key is leaked, rotate it and log the incident.

## Minimum sections
- Credential class
- Platform
- Owner
- Storage location
- Backup or recovery method
- Rotation rule
- Last review date
- Incident notes

## Plain-language rule
Explain security steps simply enough that even a beginner can follow them safely.

## Quality law
Correct. Coherent. Clear. Precise.
