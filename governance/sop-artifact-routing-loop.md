# SOP Artifact Routing Loop

## Purpose
Define the permanent operating loop for every serious artifact going forward.

## Core loop
Every serious artifact must move through:
1. Create
2. Clarify
3. Govern
4. Route
5. Implement
6. Review

## Serious artifact rule
A serious artifact includes:
- doctrines
- PDFs
- books
- key prompts
- workflows
- governance documents
- system maps
- operating standards
- important build notes
- canon-level outputs

## Routing rule
Every serious artifact must be assigned:
- a project home
- a live home
- an owner
- a next action
- a review rhythm

## Project homes
- EVE = governance, approval, operating authority
- PTR or Peter = workshop, founder doctrine, refinement proposals
- Nubeing 9 Studios = business, production, website, services, public deployment
- NeoSophia = canon, mythos, worldbuilding, symbolic systems, seeker experience
- Archive = preserved but inactive

## Actionability rule
Nothing serious should remain:
- floating
- duplicated
- unnamed
- unowned
- without next action
- without review rhythm
- purely theoretical

## Closure rule
Every major document, book, PDF, or explanation must end with a plain-language breakdown simple enough that a five-year-old could understand.

## Quality law
Correct. Coherent. Clear. Precise.

## Simple version
We do not just make things anymore.
We make them, name them, place them, use them, and review them.
